## 🚀 Overview

**Netpulse** (formerly KeepAlive) is a production-grade SaaS monitoring and network intelligence ecosystem. It combines ultra-fast uptime heartbeat monitoring with high-resolution global network diagnostics, including multi-region DNS benchmarking, geo-clustered latency analysis, and security auditing.

Built for modern infrastructure, it ensures your services—from Supabase databases to complex APIs—stay active, performant, and visible globally.

---

## 🛠️ Main Modules & Features

### 📡 Network Intelligence System

* **Global DNS Benchmark**: Real-time performance testing against major resolvers (Google, Cloudflare, Quad9, etc.) from multiple global nodes.
* **Geo Clusters**: Distributed monitoring across Asia, Europe, and the Americas to identify region-specific routing bottlenecks.
* **Security Insight**: Automated SSL/TLS health checks and security protocol auditing.
* **Data Lab**: Experimental dashboard for AI-driven network trend analysis and predictive latency mapping.

### 🛡️ Professional Uptime Monitoring

* **Hyper-Scale Heartbeats**: Multi-region pings every 30s to 1h.
* **Supabase Keep-Alive**: Specialized worker logic to prevent Supabase projects from pausing due to inactivity.
* **Smart Alerts**: Instant notifications via Email, Discord, and Slack with intelligent false-positive filtering.
* **Incident Management**: Track and resolve downtime with historical incident logging.

### 📊 Status & Transparency

* **Public Status Pages**: Customizable, premium-styled pages to showcase system reliability to your customers.
* **Leaderboards**: Visual rankings of top-performing DNS providers and network nodes.
* **Live Tickers**: Real-time activity feeds of global network pulses.

## 🚦 Getting Started

### Prerequisites

* Node.js (v18+)
* Supabase Project URL & Keys
* Firebase Account (for specific service modules)

### Installation

1. **Clone the Repository**

    ```bash
    git clone https://github.com/abir2afridi/KeepAlive.git
    cd KeepAlive
    ```

2. **Install Dependencies**

    ```bash
    npm install
    ```

3. **Environment Configuration**
    Create a `.env` file in the root directory:

    ```env
    VITE_SUPABASE_URL=your_supabase_url
    VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
    ```

4. **Run Development Server**

    ```bash
    npm run dev
    ```
